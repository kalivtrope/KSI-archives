Řešení

Úlohu budeme řešit metodou dynamického programování. Bohužel nám nepomůže si text jen rozdělit na jednotlivá slova podle mezer a pro každé takové slovo spočítat vzdálenost od hledaného slova. Tímto způsobem totiž nenajdeme slovo, které jsme si vepsali do jiného slova. Například ve větě “Toto je věahojta.” je vepsané slovo “ahoj”, které bychom pouhým rozdělením textu podle mezer nenašli.

Na začátek uveďme několik úvah:

  1. Nemá smysl hledat slova, která jsou od hledaného slova vzdálenější než prázdné slovo, protože každý řetězec obsahuje prázdné slovo jako svůj podřetězec.

  2. Řekněme, že nejbližší slovo ke hledanému slovu začíná na indexu aa a končí na indexu bb.

  3. Řekněme, že NN je délka textu (seminární práce) a SS je délka hledaného slova.

  4. Každé slovo delší než 2S2S je vzdálenější od hledaného slova více než prázdné slovo, protože musíme přidat více znaků než bychom museli odebrat do prázdného slova. (Podmínkou tohoto tvrzení je, že odebírání i přidávání znaků má stejnou hodnotu.)

  5. Máme celkem N⋅2SN⋅2S možností, které podřetězce seminární práce můžeme považovat za hledané slovo.

  6. Nyní se nabízí varianta projít všechna možná slova a pro každé z nich spočítat Levenštejnovu vzdálenost od hledaného slova. To je ale poněkud triviální a časově složité řešení, nemyslíte? Toto řešení vyžaduje N⋅2SN⋅2S krát počítat Levenštejnovu vzdálenost, tedy výsledná časová složitost takového řešení je O(N⋅S⋅S2)=O(N⋅S3)O(N⋅S⋅S2)=O(N⋅S3). Podle zadání bychom ale rádi dosáhli časové složitosti O(N⋅S)O(N⋅S).

Můžeme si všimnout, že u triviálního řešení se počítá Lenevštejnova vzdálenost na podřetězcích, které se vzájemně liší třeba jen o jeden index. Abychom nemuseli počítat zbytečně několikrát totéž, využijeme techniku dynamického programování.

Přístup k řešení technikou dynamického programování

  1. Představme si, že máme na počátku prázdný text seminární práce TT a prázdné slovo WW. V takovém případě získáme triviální výsledek na indexu 0 s Levenštejnovou vzdáleností 0.

  2. Nyní budeme postupně po jednom přidávat znaky do WW, dokud nezískáme slovo, které chceme. Výsledek bude opět triviálně na indexu 0 s Levenštejnovou vzdáleností rovnou počtu znaků ve slově WW.

  3. Pojďme si doposud získané triviální výsledky zapsat do tabulky:

empty
empty 0
a 1
b 2
c 3

Do nultého sloupce (legenda tabulky) jsem postupně připisoval písmena slova WW. První řádek reprezentuje prázdné slovo empty, druhý řádek reprezentuje přidané písmeno a atd. Na čtvrtém řádku jsme zíkali WW = "abc". Nadpis tabulky (nultý řádek) bude postupně rozkrývat text celé seminární práce TT. Nyní TT = "". Čísla v prvním sloupci reprezentují triviálně nalezenou Levenštejnovu vzdálenost pro TT = "" a SS = ""; SS = "a"; SS = "ab"; SS = "abc".

  1. Nyní do tabulky doplníme celý text seminární práce. Například TT = "deabfcg"
empty d e a b f c g
empty 0 0 0 0 0 0 0 0
a 1
b 2
c 3

Na první řádek jsme doplnili samé nuly, které jsou triviálním řešením pro libovolný text seminární práce, pokud WW je prázdné slovo (WW = "").

  1. Teď nám zbývá ta nejtěžší část, a to jak doplnit celý zbytek tabulky. Představme si, že nyní chceme doplnit Levenštejnovu vzdálenost do tabulky na souřadnice [2;2], tedy TT = "d" a WW = "a". Zrekapitulujme si prvně, co už díky tabulce víme:

    1. Ze souřadnic [1;1] víme, že pokud TT = "" a WW = "", tak WW umíme v textu TT najít s Levenštejnovou vzdáleností 0. Potom přídáme znak d do TT a znak a do WW. Přidali jsme rozdílné znaky, tedy musíme zvýšit i Levenštejnovu vzdálenost. Nahrazení znaku za jiný odpovídá přičtení jedničky podle zadání. Pokud bychom přidali stejný znak do WW i do TT, k Lenevštejnově vzdálenosti bychom nepřičítali nic. Všiměte si tedy, že nahrazení znaku za jiný odpovídá diagonálnímu pohybu v tabulce:

levDist(i,j)=levDist(i−1,j−1)levDist(i,j)=levDist(i−1,j−1)
pokud jsou znaky vložené do TT a WW totožné.

levDist(i,j)=levDist(i−1,j−1)+1levDist(i,j)=levDist(i−1,j−1)+1
pokud nejsou totožné a musíme jeden znak nahradit za jiný.

Funkce levDist() vrací Levenštejnovu vzdálenost podle zadaných souřadnic v tabulce.

b) Ze souřadnic [2;1] víme, že pokud $T$ = `"d"` a $W$ = `""`, tak $W$ umíme v textu $T$ najít s Levenštejnovou vzdáleností 0. Potom přidáme znak `a` do $W$. Přidáním znaku zvyšujeme Levenštejnovu vzdálenost o 2 podle zadání. Nyní jde o vertikální pohyb v tabulce, tedy:

levDist(i,j)=levDist(i−1,j)+2levDist(i,j)=levDist(i−1,j)+2

c) Stejným způsobem zjistíme, že odstranění znaku z $W$ odpovídá horizontálnímu pohybu v tabulce opět s Levenštejnovou vzdáleností +2, tedy:

levDist(i,j)=levDist(i,j−1)+2levDist(i,j)=levDist(i,j−1)+2

  1. Ze tří prezentovaných možností (5.a), (5.b) a (5.c) si vybereme tu, která nám zvýší Levenštejnovu vzdálenost co nejméně, tedy za (a) a výsledek zapíšeme do tabulky na souřadnice [2;2]:
empty d e a b f c g
empty 0 0 0 0 0 0 0 0
a 1 1
b 2
c 3

  1. Opakováním stejného způsobu můžeme vyplnit celou tabulku. V programu nejprve vyplním celý první sloupec, potom celý druhý sloupec atd. Za chvíli uvidíme proč. (Pokud chcete tabulku vyplnit třeba po řádcích, tak je to také možné.)

  2. Nyní máme vyplněnou celou tabulku v čase O(N⋅S)O(N⋅S).

empty d e a b f c g
empty 0 0 0 0 0 0 0 0
a 1 1 1 0 1 1 1 1
b 2 2 2 2 0 2 2 2
c 3 3 3 3 2 1 2 3

Nejzajímavější je pro nás poslední řádek tabulky, který zobrazuje Levenštejnovy vzdálenosti pro hledané slovo WW = "abc". Z tabulky už vidíme, že nejmenší Lenevštejnova vzdálenost se na posledním řádku skrývá na souřadnicích [6;4], tedy hledané slovo v textu TT s nejmenší Levenštejnovou vzdáleností 1 končí na indexu 5 (na indexu 5 se už nenachází).

  1. Nyní máme index posledního znaku hledaného slova. Nakonec si ukážeme, jak zjistit index prvního znaku. Pojďme si ale nejprve jednotlivé myšlenky přeměnit ve zdrojový kód.

Ve zdrojovém kódu si všiněme ještě jedné optimalizace paměti. V každém okamžiku nám stačí informace z posledních dvou známých sloupců tabulky. Celou tabulku tedy budeme reprezentovat jen dvěma sloupci uloženými v proměnných first a second. Po dokončení výpočtu nového sloupce obsah proměnných first a second prohodíme a do proměnné second ukládáme údaje zase z nového sloupce.

Příklad implementace

# @return posledni index prvniho nejlepsiho nalezu
def find_nearest(word, text):
    # Delka seminarni prace
    textLen = len(text)
    # Posledni radek tabulky
    levDistances = [0 for x in range(textLen)]
    # Posledni dva sloupce tabulky
    first = [0 for x in range(wordLen)]
    second = [0 for x in range(wordLen)]

    # inicializace prvniho sloupce
    # prvni radek se inicializuje automatiky propsanim nuly z prvniho sloupce
    for i in range(wordLen):
        first[i] = i + 1

    # vyplnovani tabulky
    for i in range(textLen):
        for j in range(wordLen):
            # pokud pismena souhlasi
            if(word[j] == text[i]):
                if(j != 0):
                    second[j] = first[j - 1]
                else:
                    second[j] = 0
            else:
                # vertikalni predchozi
                prevVert = 0;
                if(j != 0):
                    prevVert = second[j-1]

                # horizontalni predchozi
                prevHor = j
                if(i != 0):
                    prevHor = first[j];

                # diagonalni predchozi
                prevDiag = 0
                if(i != 0 or j != 0):
                    if(i != 0 and j != 0):
                        prevDiag = first[j - 1]
                    elif(i != 0):
                        prevDiag = 0
                    elif(j != 0):
                        prevDiag = first[j - 1]
                else:
                    prevDiag = 0

                # zjistime, ktera ze tri variant byla nejlepsi
                if(prevDiag <= min(prevVert, prevHor)):
                    # preklep, pripocitame vzdalenost 1
                    second[j] = 1 + prevDiag
                elif(prevVert < prevHor):
                    # vlozeni znaku, pripocitame vzdalenost 2
                    second[j] = 2 + prevVert
                else:
                    # smazani znaku, pripocitame vzdalenost 2
                    second[j] = 2 + prevHor

        # zapiseme nejlepsi nalezenou Levenstejnovu vzdalenost pro dany index
        levDistances[i] = second[wordLen - 1]

        # swap columns
        tmp = first
        first = second
        second = tmp

    # val = nejmensi nalezena Levestejnova vzdalenost
    # idx = index posledniho znaku, na kterem bylo nejlepsi slovo nalezeno
    val, idx = min((val, idx) for (idx, val) in enumerate(levDistances))
    return idx;

Nalezení indexu prvního písmene hledaného slova

V tabulce jsme si doposud ukládali jen informace o nejmenší Lenevštejnově vzdálenosti, ale už jsme neřešili, na kterém indexu se namezelé slovo nachází. Nalezené slovo může mít různou délku, takže teoreticky potřebujeme kromě indexu znát ještě délku slova (nebo index konce slova). Prakticky nám podle zadání stačí znát jen počáteční index.

Když se trošku víc zamyslíme nad tím, jak tabulka funguje, tak index konce nalezeného slova nám prozradí číslo sloupce v tabulce. S počátečním indexem nalezeného slova je to trošku složitější.

Když si budeme během vytváření tabulky pro každou buňku pamatovat kromě Levenštejnovy vzdálenosti i aktuální délku slova, budeme mít už kompletní informaci k úspěšnému sestavení celého řešení. Z tabulky si vybereme nejmenší Levenstejnovu vzdálenost, ze stejné buňky přečteme i délku nalezeného slova a index posledního písmene. Po odečtení délky slova získáme index prvního písmene hledaného slova.

Kompletní implementace celého algoritmu

# @return index prvniho nejlepsiho nalezu
def find_nearest(word, text):
    wordLen = len(word)
    textLen = len(text)
    levDistances = [0 for x in range(textLen)]
    foundWordLen = [0 for x in range(textLen)]
    first = [0 for x in range(wordLen)]
    second = [0 for x in range(wordLen)]
    firstWordLen = [0 for x in range(wordLen)]
    secondWordLen = [0 for x in range(wordLen)]

    # inicializace prvniho sloupce
    for i in range(wordLen):
        first[i] = i + 1
        firstWordLen[i] = i + 1

    # vyplnovani tabulky
    for i in range(textLen):
        for j in range(wordLen):
            # pokud pismena souhlasi
            if(word[j] == text[i]):
                if(j != 0):
                    second[j] = first[j - 1]
                else:
                    second[j] = 0
            else:
                # vertikalni predchozi
                prevVert = 0;
                if(j != 0):
                    prevVert = second[j-1]

                # horizontalni predchozi
                prevHor = j
                if(i != 0):
                    prevHor = first[j];

                # diagonalni predchozi
                prevDiag = 0
                if(i != 0 or j != 0):
                    if(i != 0 and j != 0):
                        prevDiag = first[j - 1]
                    elif(i != 0):
                        prevDiag = 0
                    elif(j != 0):
                        prevDiag = first[j - 1]
                else:
                    prevDiag = 0

                if(prevDiag <= min(prevVert, prevHor)):
                    # preklep, pripocitame vzdalenost 1
                    second[j] = 1 + prevDiag
                    secondWordLen[j] = firstWordLen[j]
                elif(prevVert < prevHor):
                    # vlozeni znaku, pripocitame vzdalenost 2
                    second[j] = 2 + prevVert
                    secondWordLen[j] = secondWordLen[j-1] + 1
                else:
                    # smazani znaku, pripocitame vzdalenost 2
                    second[j] = 2 + prevHor
                    secondWordLen[j] = firstWordLen[j] - 1

        # zapiseme nejlepsi nalezenou Levenstejnovu vzdalenost pro dany index
        levDistances[i] = second[wordLen - 1]
        # zapiseme delku nejlepsiho slova na danem indexu
        foundWordLen[i] = secondWordLen[wordLen - 1]

        # swap columns
        tmp = first
        first = second
        second = tmp
        
        tmp = firstWordLen
        firstWordLen = secondWordLen
        secondWordLen = tmp

    # val = nejmensi nalezena Levestejnova vzdalenost
    # idx = index, na kterem bylo nejlepsi slovo nalezeno
    val, idx = min((val, idx) for (idx, val) in enumerate(levDistances))
    return idx - foundWordLen[idx] + 1;
