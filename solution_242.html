Popis struktury podproblémů

Označme počet řádků vesmírné dráhy jako nn, počet sloupců mm a počáteční pozici vesmírné lodi (0, j0j0). (Vesmírná loď tedy začíná na řádku 0 a končí na řádku n-1.) Dále ako AijAij označme pravdivostní hodnotu udávající, zda je v i-tém řádku a j-tém sloupci asteroid.

Potřebujeme vypočítat nejkratší čas, za který se vesmírná loď může ze své startovní pozice dostat do i-tého řádku (0..n-1), j-tého sloupce (0..m-1), ve směru dd (-1, 0, 1) a v rychlosti ss (1, 2, 3); označme tento optimální čas TijdsTijds. Hodnota stavů, do kterých se nelze vůbec dostat (např. jsou mimo hranice dráhy nebo na asteroidu), nechť je ∞∞.

Zdůvodnění korektnosti rekurentní rovnice

Bázové případy (tj. první tři řádky) vychází přímo ze zadání. V ostatních případech lze optimální trasu do stavu (i, j, d, s) rozložit na dva úseky: cestu do předposledního stavu (i-1, j-d, d’, s’) a krok z tohoto předposledního stavu do stavu (i, j, d, s). Množina PijdsPijds obsahuje právě ty předchozí stavy, které jsou povolené zadáním. Pokud má být čas do (i, j, d, s) minimální možný, tak musíme zvolit nejkratší možnou cestu do předposledního stavu – kdybychom totiž do předposledního stavu neletěli optimálně, pak by bylo možné tento úsek cesty zrychlit a tím snížit celkovou dobu letu. Pro všechny možné předchozí stavy vypočítáme součet (1) času přeletu do předchozího stavu a (2) času potřebného k přechodu z předchozího stavu do aktuálního stavu. (Čas (2) je podle zadání délka děleno rychlost.) Z těchto časů vybereme minimum.

Algoritmus pro nalezení hodnoty minimálního času

Protože se podproblémy překrývají, použijeme dynamické programování. K výpočtu optimálního času do stavu (i, j, d, s) potřebujeme pouze časy pro (některé) ze stavů z řádku i-1. Budeme tedy postupovat od řádku s nejmenším indexem po řádek s nejvyšším indexem – tím zajistíme, že všechny potřebné podproblémy budeme mít vypočítané, když je budeme potřebovat. Výsledky budeme ukládat do čtyřrozměrného pole hodnot. (Technický detail: při implementaci v Pythonu je trochu jednodušší použít slovník indexovaný čtveřicemi (i, j, d, s).)

Alternativním přístupem by bylo použít Dijkstrův algoritmus (příp. v kombinaci s heuristikou, A*) pro hledání nejkratší cesty v grafu. Dijkstrův algoritmus je podobný přístupu pomocí dynamického programování, prochází však stavy v jiném pořadí: vybere vždy hladově ještě neprozkoumaný stav, do kterého se lze dostat ze startu nejrychleji. Výběr dalšího stavu tím ale neprobíhá v konstantním čase a celková asymptotická složitost Dijkstrova algoritmu je kvůli tomu mírně větší.

Analýza časové složitosti

Časové složitosti algoritmu dominuje vyplnění čtyřrozměrné tabulky rozměrů n×m×3×3n×m×3×3. Výpočet každého políčka má konstantní složitost, časová složitost algoritmu je tedy O(n⋅m)O(n⋅m).

Analýza prostorové složitosti

Popsaný algoritmus si pamatuje optimální časy všech stavů (a v případě rekonstrukce trasy také jejich předchůdce), takže i prostorová složitost je O(n⋅m)O(n⋅m). Tu lze však vylepšit na O(m)O(m), když si uvědomíme, že si stačí pamatovat časy pro stavy na jednom předchozím řádku (to nám však pomůže pouze v případě, že nechceme rekonstruovat trasu).

Rozšíření o rekonstrukci optimální cesty

Rekonstrukci trasy lze provádět několika způsoby. Asi nejjednodušší je už při výpočtu optimálních časů ukládat předchůdce, ze kterých je nejlepší se do daného stavu dostat (tj. ve kterém stavu se realizovalo minimum rekurentního vztahu). Zrekonstruovat cestu pomocí této informace je pak triviální: začneme v posledním stavu trasy a postupujeme vždy o krok zpět, dokud nedojdeme do počáteční stavu na řádku 0. Rekonstrukce má tedy časovou složitost O(n)O(n).

Implementace algoritmu

from collections import defaultdict
from math import inf


def compute_fastest_trajectory(start, asteroids):
    """Compute shortest path from the start to the last row.

    Args:
        start: Column where the spaceship starts.
        asteroids: 2D array of boolean values indicating asteroids.

    Return:
        tuple of optimal time, optimal trajectory,
        where the trajectory is a list of (row, col, direction, speed).
    """
    n, m = len(asteroids), len(asteroids[0])
    best_times = defaultdict(lambda: inf)
    best_times[0, start, 0, 0] = 0
    best_predecessors = {}
    for row in range(1, n):
        for col in range(m):
            if asteroids[row][col]:
                continue
            for d in [-1, 0, 1]:
                for speed in range(1, 4):
                    time, predecessor = find_best_time_and_predecessor(
                        best_times, row, col, d, speed)
                    best_times[row, col, d, speed] = time
                    best_predecessors[row, col, d, speed] = predecessor
    best_time, state = min(
        (best_times[n-1, c, d, s], (n-1, c, d, s))
        for c in range(m) for d in [-1, 0, 1] for s in range(1, 4))
    # Trajectory reconstruction
    reversed_trajectory = [state]
    for i in range(n-1):
        state = best_predecessors[state]
        reversed_trajectory.append(state)
    trajectory = list(reversed(reversed_trajectory))
    return best_time, trajectory


def find_best_time_and_predecessor(best_times, row, col, direction, speed):
    distance = 1 if direction == 0 else 1.4
    time_delta = distance / speed
    candidates = []
    for prev_state in get_possible_previous_states(row, col, direction, speed):
        prev_time = best_times[prev_state]
        candidates.append((prev_time, prev_state))
    best_prev_time, best_prev_state = min(candidates)
    final_time = best_prev_time + time_delta
    return final_time, best_prev_state


def get_possible_previous_states(row, col, direction, speed):
    previous_states = []
    for speed_delta in [-1, 0, 1]:
        prev_state = (row-1, col-direction, direction, speed - speed_delta)
        previous_states.append(prev_state)
    if speed == 1:
        for prev_direction in [-1, 0, 1]:
            prev_state = (row-1, col-direction, prev_direction, speed)
            previous_states.append(prev_state)
    return previous_states
