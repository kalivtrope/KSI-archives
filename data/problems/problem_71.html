Zadání

Bylo krásné prázdninové odpoledne a tučňák Karlík se rozhodl, že zavolá svým kamarádům – losovi, sobovi a krajtě (pozn. organizátorů: anglicky je to python) – a půjdou si společně hrát ven. Za půl hodiny se všichni čtyři sešli na hřišti. Rozhodli se, že si zahrají svoji oblíbenou hru. Karlík donesl křídy a na celé hřiště namaloval pravidelnou čtvercovou síť. Ostatní přispěli něčím dobrým na zub a několik laskomin rozmístili na náhodná políčka. Potom hra začala.

Tvým úkolem bude co nejpřesněji a nejefektněji naimplementovat chování čtyř zvířátek (tučňák, los, sob a krajta). Zvířátka dokáží provádět dvě činnosti – jít (go()) a jíst(eat()). Chovají se však odlišně.

Každé zvířátko má přesně daná pravidla, jak se může pohybovat. Ty ti ukáží naši čtyři kamarádi – tučňák Karlík, sob Rudolf, los Alois a krajta Agáta. Do svého kódu si je načteš z naší knihovničky – animals_visualiser.pyc. (Od 9. října je v nové verzi, která řeší několik nedostatků, používej prosím tuhle.) V ní bude obsaženo vše potřebné pro řešení úlohy – vizualizace zvířátek, pomocné metody a třídy i vzorová zvířátka. Importování je jednoduché – do stejné složky jako máš soubor s řešením si uložíš animals_visualiser.pyc a na začátku souboru s řešením zadáš například from animals_visualiser import Playground, nebo from animals_visualiser import PenguinKarel. Zobrazování hřiště ti vysvětlíme na konci zadání. Aby ti úloha správně fungovala, budeš potřebovat doinstalovat dva pythonní moduly – jak na to je úplně na konci zadání. Pokud budeš mít potíže s jeho instalací, neváhej se nám ozvat v diskuzi a my ti rádi pomůžeme.

Tvé řešení bude obsahovat u každého zvířátka implementaci obou metod včetně slovního popisu a zdůvodnění. Krajta (Python) bude navíc potřebovat i funkci vracející seznam s poli, na kterých je ocas (od těla ke konci). Kostra řešení pro tučňáka bude vypadat následovně:

class Penguin(Animal):
    def __init__(self, field, orientation, name):
        Animal.__init__(self, field, orientation, name)
        
    def go(self):
        #Tva implementace chozeni

    def eat(self):
        #Tva implementace jezeni

Část class Penguin(Animal) značí, že vytváříme třídu Penguin, která rozšiřuje chování třídy Animal, která vypadá následovně:

class Animal:
    def __init__(self, field, orientation, name):
        self.field = field
        self.orientation = orientation
        self.name = name

    def go(self):
        pass

    def eat(self):
        pass

Jediné, co v ní najdeme užitečného, je, že si ukládá aktuální políčko a směr otočení při vytvoření zvířátka. Také zvířátko pojmenuje. Při jakékoliv činnosti nedělá nic (pass). Třída Pinguin tzv. dědí z třídy Animal. Při vytváření zvířátka nesmíme zapomenout volat konstruktor třídy, kterou rozšiřujeme. Proto máme takový prapodivný konstruktor:

def __init__(self, field, orientation, name):
    Animal.__init__(self, field, orientation, name)

Problematika dědění není složitá, ale u této úlohy se jí nemusíš zabývat, stačí, když použiješ kostru řešení. Při psaní kódu bys měl uvést za názvem třídy do závorky Animal a v souboru s řešením mít i třídu Animal (nebo ji importovat). Všechna tebou vytvořená zvířata tak budou při vytváření potřebovat startovní políčko, orientaci a jméno. Ty můžeš samozřejmě tyto proměnné/atributy využívat. U každého zvířátka si můžeš ukládat i jiné informace, ale musíš vždy vyplnit orientaci, políčko a jméno, na kterém se zvířátko nachází. Atributy jistě už používat umíš, tady je příklad:

# Vytvoříme Karlíka, který bude stát na novém travnatém poli
travnatePolicko = Field(FieldType.GRASS, FoodType.NOTHING)
karlik = Penguin(travnatePolicko, Orientation.TOP, 'Karlik')

# Otočka dolů.
karlik.orientation = Orientation.BOTTOM

# Jdeme se koupat
poleSVodou = Field(FieldType.WATTER, FoodType.NOTHING)
karlik.field = poleSVodou

Objekt Field a výčtové typy FieldType a Orientation si vysvětlíme za chvilku. Tady stačí vědět, že karlik.field = novepolicko posílá Karlíka na nové políčko a karlikovoPole = karlik.field nám do proměnné karlikovoPole uloží pole, na kterém Karlík stojí.

Konstruktory si pro jednodušší práci můžeš doplnit o další nepovinné argumenty – nemusí se při volání uvést a určíš je tvarem funkce(povinny_argument, nepovinny_argument=defualtni_hodnota, například takto můžeme tučňáka při vytváření obarvit:

def __init__(self, field, orientation, color='bila'):
        Animal.__init__(self, field, orientation)
        self.color = color

Pokud bys chtěl nějaké funkce použít u všech zvířátek, uveď je do těla třídy Animal, všechny zvířátka z této třídy dědící je budou moci využít. Tímto můžeš svou práci velmi zjednodušit. Například si můžeme definovat funkci double_go:

class Animal:
    def __init__(self, field, orientation, name):
        self.field = field
        self.orientation = orientation
        self.name = name

    def go(self):
        pass

    def eat(self):
        pass
        
    def double_go(self):
        self.go()
        self.go()

U libovolného potomka můžeme funkci double_go využít:

karlik = Penguin(travnatePolicko, Orientation.TOP, 'Karlik')
karlik.double_go()

Funkci můžeš samozřejmě využít i při implementaci metod u jednotlivých zvířátek.

Kromě třídy Animal budeš potřebovat ještě několik dalších pomocných tříd, všechny najdeš v přiložené knihovničce (animals_visualiser.pyc). Okomentujeme si je:

Políčka jsou charakterizována objektem typu Field definovaného následovně:

class Field:
    """
    Charakterizuje jednotlive pole,
    ktere ma svuj typ a pole s potravou.
    """

    def __init__(self, typ, food):
        self.__type = typ
        self.food = food
        self.left = None
        self.right = None
        self.top = None
        self.bottom = None

    def get_type(self):
        return self.__type

    def __str__(self):
        return '{} <{}>'.format(str(self.__type), str(self.food))

Atributy left, right, top a bottom jsou ukazateli na vedlejší políčka. Nejspíš je nebudeš muset upravovat. Používají se následovně:

travnatePolicko = Field(FieldType.GRASS, FoodType.NOTHING)
poleSVodou = Field(FieldType.WATTER, FoodType.NOTHING)
travnatePolicko.left = poleSVodou

# Druhá strana sousedského vztahu se nedoplní sama
print(poleSVodou.right)
    None
# Musíme ji určit sami
poleSVodou.right = travnatePolicko

Typ políčka je tzv. privátní – můžeme jej použít pouze uvnitř dané třídy. Privátní atribut vytvoříme tak, že zadáme dvě podtržítka před jméno atributu. Abychom typ políčka mohli číst v kódu mimo třídu, vytvořili jsme si metodu get_type(). Typ políčka je tak jen pro čtení. Nemůžeme do něj zvenčí nic zapsat.

A jako poslední si vysvětlíme výčtové typy. Jde o náš nový typ, který může nabývat pouze našich předem nadefinovaných hodnot. Například typ Orientation se definuje a používá následovně:

# Musíme importovat potřebnou knihovnu
from enum import Enum

# Vytvoříme nový výčtový typ
Orientation = Enum('TOP', 'LEFT', 'BOTTOM', 'RIGHT')
# Typ políčka je taky výčtový typ
FieldType = Enum('OUT_OF_MAP', 'GRASS', 'WATTER', 'WALL')
# Stejně jako typ potravy:
FoodType = Enum('NOTHING', 'TOXIC_MUSHROOM', 'EDIBLE_MUSHROOM', 'LICHEN', 'FISH')

travnatePolicko = Field(FieldType.GRASS, FoodType.NOTHING)
karlik = Penguin(travnatePolicko, Orientation.TOP, 'Karlik')
karlik.orientation = Orientation.BOTTOM

Vizualizace

Pro zobrazení chování čtyři kamarádů i svých implementací ti přijde vhod nějaký hrací plánek pro zvířátka, který si budeš moct zobrazit.

Vše potřebné je opět v knihovničce (animals_visualiser.pyc). Základem je třída Playground, která reprezentuje hrací pole, umožňuje přistupovat k jednotlivým políčkům (get_field(x,y)). Také dokáže a zobrazit hrací pole (show()) se zvířátky, které si na hřiště přidáš (add_animal(animal)). Tuto metodu můžeš použít jak pro své objekty, tak pro instance vzorových zvířátek. Pro přidání našich vzorových zvířátek máš ve třídě Playground zkratky (create_penguin_karel(x,y,orientation), create_reindeer_rudolf(x,y,orientation), create_moose_alois(x,y,orientation) a create_python_agata(x,y,orientation)). Tyto metody přidají zvířátko na hřiště a zároveň ti je vrátí, aby si jím mohl pohybovat. Pro vytvoření hřiště potřebuješ dvojrozměrné pole objektů typu Field, pokud se ti nechce vytvářet, využij náhodné hřiště, které ti vytvoří funkce create_random_playground(width,height). Knihovnička obsahuje další užitečné metody a objekty, rozumné vývojové prostředí ti je zobrazí. Na závěr by se ti mohl hodit nějaký příklad:

# Importování
from animals_visualiser import create_random_playground, ReindeerRudolf


# Tvorba nového hřiště
hriste = create_random_playground(20, 10)
    
# Přidáme Karla na hřiště
karel = hriste.create_penguin_karel()
    
# karel udělá sto kroků
for i in range(100):
    karel.go()

# Můžeme si ale nejdřív zvířátko vytvořit (stojí v levém horním rohu)
rudolf = ReindeerRudolf(hriste.get_field(0,0),Orientation.TOP)
    
# Pokud jej chceme vykreslit, musíme jej navíc dát na hřiště
hriste.add_animal(rudolf)

# Rudolf se nají a udělá krok
rudolf.eat()
rudolf.go()

# Obdobně můžeš přidat své zvířátko:
richard = Reindeer(hriste.get_field(0,0),Orientation.TOP)
hriste.add_animal(richard)


# Hřiště si můžeme kdykoliv vykreslit:
hriste.show()

# nebo uložit jako obrázek:
hriste.save()

# Defaultně se uloží jako "zvirata.png" do aktuálního adresáře.
# Můžeš volit i jinou cestu (typ se určí podle koncovky):
hriste.save("/home/pepa/pictures/animials.png")
hriste.save("/home/pepa/pictures/animials.jpeg")

Na windows můžeš mít problém s funkcí show – pokud se nedaří vykreslený obrázek otevřít z důvodu neexistujícího souboru, zkus dát do kódu příkaz pro čekání, aby interpretr hned neskončil a tvůj obrázek nesmazal:

# Import modulu pro spánek
import time

hriste = create_random_playground(20, 10)

hriste.show()

# Půl minuty program spí
time.sleep(30)

Může jít ale i o problém s přístupem k dočasným souborům, v takovém případě doporučujeme obrázky ukládat jinam (např.: hriste.save("zvirata.bmp")) a otevřít pomocí běžného programu na obrázky.

Implementaci chování všech čtyř zvířátek nám napiš do jednoho souboru – jako kostru použij soubor kostra-reseni.py. V druhém souboru popiš, jak jsi na implementaci přišel a zdůvodni, proč tvé řešení funguje. Nezapomeň odevzdat oba soubory.

Instalace chybějících modulů

Pro chod vykreslovacího modulu jsou potřeba dva moduly, které nejsou součástí běžné instlace pythonu – enum a pillow. Jejich doinstalování by nemělo být složité a postup se ti jistě bude do budoucna hodit.

Pro instalování pythonních modulů slouží prográmek pip – ve windows verzi by měl být součástí instalace python2.7 od verze 2.7.9. Pod linuxem bude nejspíš třeba pip doinstalovat. Nainstalovat jej můžeš přes svůj oblíbený balíčkovací systém pod názvem python-pip, například jednoduchým příkazem v terminálu – pro ubuntu/linux mint bude příkaz vypadat takto: sudo apt-get install python-pip. Jistě jsi ale už nějaký program nainstaloval, takže víš, jak na to.

Pokud máme pip nainstalovaný, tak stačí otevřít příkazovou řádku/terminál (ne shell pythonu). Do terminálu zadáme příkaz pip install enum a pro druhý modul podobně pip install pillow. Pokud vše proběhlo bez chyb, máš vyhráno a můžeš se pustit do programování. Pro kontrolu zkus zadat do interpretru import enum a import PIL.Image. Pokud proběhnou příkazy bez chyb, měl by ti funovat i modul pro vykreslování.

Pokud máš nainstalovaný i python3.5, dej si pozor, jestli spouštíš správný pip – pip2 je pro python2.7, pip3 pro python3.5.

Pokud došlo k nějaké chybě, tak zkus nejprve update pipu příkazem pip install --upgrade pip, pokud je chyba s přístupovými právy, zkus powershell/cmd spustit jako správce, nebo na linuxu spustit příkazy s právy roota – třeba takto: sudo -H pip install enum.

Pokud dostaneš chybu jenom u stažení balíku pillow, je možné, že již máš nainstalovaný modul image, který nedovolí nainstalovat pillow. Zkus jej odinstalovat pomocí pip uninstall image a pak zadat znovu příkaz k instalaci balíku pillow.

Pokud ti stále vykreslování nefunguje, napiš nám do diskuse a pokusíme se ti pomoct. Zkus zjistit, jakou verzi pythonu máš a jako chybu a kde dostáváš.

Nejen správu balíčků, ale i veškerou práci ti zjednodušší nějaké vývojové prostředí, můžeš zkusit třeba PyCharm.
