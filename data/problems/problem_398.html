Zadání

Zatím jsi se učil pracovat s databázemi především na úrovni jazyka SQL. V této úloze se naučíme, jak se připojit k databázi z tvého programu v Pythonu.

Pro připojení k databázi v Pythonu je k dispozici spousta knihoven:

  • sqlite3: snadno použitelná knihovna, která pracuje s databází uloženou pouze na tvém disku. sqlite je taková chytřejší Excelovská tabulka – můžeš nad ní dělat SQL dotazy. Pořád se však jedná o soubor uložený v počítači na disku, knihovna nepodporuje připojení k síťovému serveru.

    Zjednodušený příklad použití knihovny:

    import sqlite3

sqlite_connection = sqlite3.connect('SQLite_Python.db')
cursor = sqlite_connection.cursor()

cursor.execute("SELECT * from students")
for row in cursor.fetchall():
    print(row)

cursor.close()
sqlite_connection.close()

  • Knihovny pro přímé připojení k SQL serveru: mysql, psycopg a mnohé další. Tyto knihovny ti umožní napsat SQL dotaz, poslat ho na server a přečíst vrácenou tabulku. Zjednodušený příklad použití knihovny:

     import pymysql.cursors

 # Connect to the database
 connection = pymysql.connect(host='db.myserver.cz', user='ksi-admin', password='loskarlos', database='db', charset='utf8mb4',
                              cursorclass=pymysql.cursors.DictCursor)

 with connection:
     with connection.cursor() as cursor:
         cursor.execute("SELECT `id`, `password` FROM `users` WHERE `email`='test@test.com'")
         result = cursor.fetchone()
         print(result)

  • Knihovny řešící zároveň ORM: nejrozšířenější z těchto knihoven je sqlalchemy, o které se budeme učit v této úloze.

    Tyto knihovny poskytují na první pohled složitější přístup k databázi, v praxi se však ukazuje, že se tento složitější přístup vyplácí, protože je přehlednější a méně náchylný na chyby programátora.

    V moderních programech se prakticky nikdy nepíší přímo SQL příkazy. Knihovny jako sqlalchemy tento moderní přístup podporují. V sqlalchemy lze samozřejmě psát i čisté SQL příkazy, ale nedělá se to. Důvody si vysvětlíme níže.

Object–relational Mapping (ORM)

Jednou ze zásadních vlastnosti knihoven pro přístup k SQL je, jestli knihovny podporují ORM. Pro alespoň trochu složitější databáze se vždy používají knihovny s ORM.

ORM je mapování řádků tabulek na objekty.

Příklad

Mějme v databázi tabulku courses (jeden předmět ve škole) s těmito atributy:

  • id: varchar(255)
  • name: varchar(255)
  • description: varchar(255)
  • credits: integer

Specifikaci této tabulky odpovídá následující třída v Pythonu:

from sqlalchemy import Column, Integer, String
from sqlalchemy.ext.declarative import declarative_base

Base = declarative_base()

class Course(Base):
    __tablename__ = 'courses'

    id = Column(Integer, primary_key=True)
    name = Column(String(255))
    description = Column(String(255))
    credits = Column(Integer)

    def __repr__(self) -> str:
        return f'{self.id}: {self.name}'

Této třídě říkáme model, v našem případě model pro Course. V projektech pracujících s databázemi často bývá složka nebo soubor models, kde jsou právě definice tříd pracujícími s daty z databáze.

Zde se můžete podívat na modely pro databázi, s kterou pracujeme v celé této vlně.

Místo SQL příkazu tak můžeme psát následující kód:

for name, credits in session.query(Course.name, Course.credits):
    print(name, credits)

Všimni si, že v programu není ani jeden SQL příkaz. To přináší především následující výhody:

  • Ještě před spuštěním programu lze kontrolovat, že atributy tabulek skutečně existují, mají správný typ apod.
  • Uživatelské vstupy jsou lépe chráněny od útoků na databázi.

Knihovna vnitřně překládá příkaz programátora na SQL příkaz, který pak posílá do databáze. Ale protože to dělá knihovna a ne programátor, je tu větší šance, že bude takový překlad napsaný bezchybně a bude třeba lépe ošetřovat uživatelské vstupy nebo kontrolovat neplatné operace.

Výhodou zápisu struktury databáze pomocí tříd je také to, že lze celou databázi vytvořit na základě oné struktury. Skutečně, sqlalchemy poskytuje nástroj, který vezme definice tříd a podle nich vytvoří celou databázi.

sqlalchemy

Nyní ti ukážeme pár vybraných nejčastějších příkazů z sqlalchemy, aby sis udělal představu, jak tato knihovna funguje. Podrobná dokumentace knihovny je k dispozici zde.

Připojení k serveru

from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
engine = create_engine('mysql://username:password@localhost/mydatabase?charset=utf8mb4')
session = sessionmaker(bind=engine)()

Přes objekt session se nyní přistupuje k databázi. Představme si v následujících úlohách, že máme třídy přesně kopírující databázi popsanou v předchozích úlohách. Budeme je používat v následujících příkladech.

Vztahy mezi tabulkami, join

Mezi tabulkami je možné definovat vztahy. Pokud například chceme zapsat, že člověk má odkaz na určitou adresu skrze sloupec HomeAddress, napíšeme:

class Person(Base):
    __tablename__ = 'Person'

    id = Column('PersonID', Integer, primary_key=True)
    name = Column('Name', String(255), nullable=False)
    home_address_id = Column('HomeAddress', ForeignKey(Address.id))

    home_address = relationship(Address)

    def __repr__(self) -> str:
        return str(self.name)

Na řádku home_address_id se pomocí ForeignKey uvádí odkaz do jiné tabulky. Všimněte si, že jméno oné tabulky není řetězec, ale přímo Pythoní třída. Tady můžeme vidět přesně tu kontrolu před spuštěním, kterou jsme si popisovali v začátku úlohy. Kdyby řádek byl například home_address_id = Column('HomeAddress', ForeignKey(Addresss.id)), můžeme tuto chybu odhalit ještě před spuštěním programu.

Tento řádek nám umožní položit dotaz s využitím příkazu join: chceme u všech lidí vypsat jejich města.

>>> session.query(Person, Address.city).join(Address, Person.home_address_id == Address.id).all()
[(Karlík, 'Brno'), (Los Karlos, 'Vysoké Tatry'), (Sob Karsob, 'Trondheim'), (Želva Julie, 'Brno')]

join tedy funguje tak, že stávající tabulku (ze začátku první tabulku v atributu metody query – Person) začne spojovat s tabulkou, která byla předána metodě join jako argument.

Příkaz si můžeme zkrátit, protože definice třídy sama ví, podle kterých sloupců má provést join:

>>> session.query(Person, Address.city).join(Address).all()

Kromě definice sloupů tabulky je ve třídě Person také home_address, což je tzv. definice vztahu. Jedná se o pokročilejší techniku, kterou v této úloze nebudeme příliš využívat.

Knihovna samozřejmě poskytuje i další verze joinů, např. outerjoin

Další užitečné příkazy

Další typy příkazů

Knihovna sqlalchemy kromě hledání podporuje samozřejmě i přidávání prvků a jejich mazání.

Příklady:

semester = Semester(name='2021/22 Jaro')  # standard python class instance
session.add(semester)
session.query(User).filter(User.age < 18).delete()
Tato úloha již byla uzavřena a není možné ji odevzdat. Navazující úlohy ti však byly automaticky odemčeny. Navíc se můžeš podívat na vzorové řešení úlohy.
