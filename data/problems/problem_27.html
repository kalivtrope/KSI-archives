Zadání

Mnoho problémů lze řešit tak, že nejprve vyřešíme zjednodušenou verzi problému a řešení zjednodušeného problému rozšíříme na řešení původního problému. Této technice se říká zmenši a panuj, anglicky decrease-and-conquer. Může být implementována buď iterativně (pomocí cyklu) nebo rekurzivně (funkce s jediným rekurzivním voláním). Příkladem může být řešení faktoriálu n!=n⋅(n−1)⋅…⋅2⋅1n!=n⋅(n−1)⋅…⋅2⋅1. Srovnej rekurzivní a iterativní verzi:

def factorial_rec(n):
    if n == 0:
        return 1
    else:
        return n * factorial_rec(n - 1)

def factorial_iter(n):
    result = 1
    for k in range(1, n + 1):
        result *= k
    return result

Promysli si, kde u iterativní verze dochází k onomu zmenšování problému. Oproti rekurzivnímu algoritmu je to méně vidět, ale ke zmenšování problému dochází – po každém průchodu cyklem mi stačí spočítat o jeden součin méně.

Řazení

Podívejme se nyní na problém řazení čísel v seznamu. Jednoduchý přístup k řešení je brát čísla zleva doprava a postupně je zařazovat (zleva budujeme uspořádanou posloupnost, kterou s každým novým zařazeným číslem prodlužujeme o 1). Po každém kroku nám zbývá zařadit o 1 číslo méně. Tomuto algoritmu se říká řazení vkládáním (anglicky Insert sort).

def insert_sort(list):
    for i in range(1, len(list)):
        key = list[i]
        # Vloz key (list[i]) do serazene posloupnosti list[0 .. i - 1]
        j = i
        while j > 0 and list[j - 1] > key:
            list[j] = list[j - 1]
            j -= 1
        list[j] = key

Zajímavým cvičením je zkusit si napsat rekurzivní variantu řazení vkládáním. Rozmysli si, jaká je složitost tohoto algoritmu.

V nejhorším případě potřebujeme posunout v každé iteraci for-cyklu všechny prvky již seřazené posloupnosti (plus ten, který zrovna zařazujeme), tj. 2 v první iteraci, 3 ve druhé iteraci až n v poslední iteraci. Celkem je tedy těchto přesunů 2+3+...+n=(n+2)(n+1)/2=O(n2)2+3+...+n=(n+2)(n+1)/2=O(n2). Složitost algoritmu je tedy kvadratická.

Rychlost zmenšování

Nejčastější variantou techniky Zmenši a panuj je zmenšení (redukce) na problém o 1 menší (příkladem je jak výpočet faktoriálu, tak řazení vkládáním), někdy je však možné zmenšit velikost vstupu na polovinu a zmenšování může být dokonce i nepravidelné. Příkladem známého algoritmu, při kterém dochází k zmenšování problému na polovinu, je binární vyhledávání. Příkladem algoritmu, při kterém dochází k poměrně nepravidelnému zmenšování problému, je Euklidův algoritmus pro výpočet největšího společného dělitele dvou čísel. Ten je založen na pozorování, že nsd(a,b)=nsd(bmoda,a)nsd(a,b)=nsd(bmoda,a) pro a>0a>0.

def nsd(a, b):
    if a == 0:
        return b
    else:
        return nsd(b % a, a)

print nsd(36, 42)

Na rozdíl od faktoriálu zde již není 1 k 1 korespondence s obvyklou definicí největšího společného dělitele. Korektnost funkce tady vyplývá z korektnosti zjednodušovacího rekurzivního kroku a bázového případu (nsd(0,b)=bnsd(0,b)=b), ke kterému výpočet nevyhnutelně směřuje (a v konečném počtu kroků se k němu dostane).

Zkus si napsat iterativní verzi Euklidova algoritmu (můžeš pak porovnat se vzorovým řešením kliknutím na následující rozbalovací panel).

Iterativní verze 
def nsd(a, b):
    while a > 0:
        a, b = b % a, a
    return b
Tato úloha již byla uzavřena a není možné ji odevzdat. Navazující úlohy ti však byly automaticky odemčeny. Navíc se můžeš podívat na vzorové řešení úlohy.
