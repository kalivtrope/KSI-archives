Zadání

Karlík pro svou oblíbenou kavárnu programoval systém na udělování slev zákazníkům. Ředitel Cofi-Cofi rozhodl, že drobnou slevu dostane každý zákazník, jehož číslo objednávky je dělitelné 2 (tedy je sudé) a nebo je dělitelné 5. Karlík si na to napsal pomocnou funkci a aby se mu to lépe debugovalo, tak si do nich přidal pomocný výpis.

def is_x_divisible_by_y(x, y):
    print(f"[debug] Test zda je cislo {x} delitelne {y}.")
    return x % y == 0

Následně si to zkusil a odpověď byla správná, zaskočila ho ale jedna věc - vypsal se mu pouze jeden z testovacích výpisů. Python ověřil, že číslo 10 je dělitelné 2 a vůbec neřešil, zda je dělitelné 5.

>>> is_x_divisible_by_y(10, 2) or is_x_divisible_by_y(10, 5)
[debug] Test zda je cislo 10 delitelne 2.
True

Co je to za magii, říkal si Karlík. A jelikož je to zvědaví tučňák, tak si hned chtěl zkusit, zda je to opakovatelné chování a zda se to tak funguje jenom pro logické nebo (or), nebo i pro logické a (and).

Or example

def or_example(x):
    return is_x_divisible_by_y(x, 2) or is_x_divisible_by_y(x, 5)
>>> or_example(1)
[debug] Test zda je cislo 1 delitelne 2.
[debug] Test zda je cislo 1 delitelne 5.

>>> or_example(2)

[debug] Test zda je cislo 5 delitelne 5.
True

>>> or_example(10)

def and_example(x):
    return is_x_divisible_by_y(x, 2) and is_x_divisible_by_y(x, 5)

Jak vidíš z příkladů výše, Python je, stejně jako většina dalších programovacích jazyků, chytrý a vyhodnocuje podmínky lenivě, čímž může zrychlit vyhodnocení programu, pokud tvé podmínky obsahují or nebo and operátor. Vyhodnocuje je výrazy zleva, a když po vyhodnocení části výrazu zjistí, že už teď je odpověď jasná, tak se neobtěžuje s vyhodnocováním zbytku výrazu a rovnou odpoví.

V tabulkách výše vidíš pravdivostní hodnoty pro operátory or a and. Tučně zvýrazněné jsou části, které Python vyhodnotí. Kurzívou jsou ty, na kterých při vyhodnocování zleva nezáleží.

Iterátory, generátory, range

Když to Karlík zjistil, hned zkoušel googlit, kde dál umí být Python líný a tím pádem dělat věci efektivněji. Zjistil, že Python má něco, čemu se říká iterátory a generátory. Vysvětlíme ti je v následujících úlohách, pro teď si ale jen ukažme na jednoduchém příkladu, jak efektivní mohou být.

Použijeme na to funkce any a all, které jsou integrované v Pythonu a dělají přesně to, co jejich název napovídá. Pokud si nejsi jistý, mrkni na ně do následujícího bloku.

Funkce any a all

Jsou naimplementovány nějak takto:

def any_example(something):
    for x in something:
        if x:
            return True
    return False

print(any_example([])) # False
print(any_example([False])) # False
print(any_example([False, True])) # True
print(any_example([0])) # False
print(any_example([0, 1])) # True

def all_example(something):
    for x in something:
        if not x:
            return False
    return True

print(all_example([])) # True
print(all_example([False])) # False
print(all_example([False, True])) # False
print(all_example([True, True])) # True
print(all_example([0])) # False
print(all_example([0, 1])) # False
print(all_example([1, 2])) # True

Oba následující programy vyhodnocují stejný vstup a vracejí stejné odpovědi, levý na to však potřebuje pár bytů paměti a stihne to téměř okamžitě, a pravý potřebuje přes půl GB, tedy více než 17 milionkrát více, a potrvá několik sekund až minut.

import sys


# the following command finishes immediately
print(sys.getsizeof(range(100000000))) # 48 (48 bytes)

# Range evaluation example
print(any(range(100000000))) # True
print(all(range(100000000))) # False
 
import sys

# the following command takes roughly 8 seconds
evaluated_list_example = [i for i in range(100000000)]
print(sys.getsizeof(evaluated_list_example)) # 859724472 (859 724 472 bytes)

# List evaluation example
print(any(evaluated_list_example)) # True
print(all(evaluated_list_example)) # False

To je magie, pomyslel si Karlík. Chvilku se na příklady díval a pak si všiml rozdílu - první příklad vytváří proměnou, která má 48 B, druhý vytváří proměnnou, která má přes 800 MB. Ta větší proměnná mu dává smysl - Python si někam potřebuje uložit všech 100 000 000 čísel, jak to ale v prvním příkladu vmáčkne do 48 bytů? Trik je v tom, že pro range si Python uloží parametry, tzn že začíná od 0 a má jít po 100000000. Následně vždycky, když ho požádáš o prvek, tak se podívá, který ti dal poslední, a dá ti o jedna větší.

Samotné vyhodnocení pomocí any a all trvá pro oba programy v podstatě stejně dlouho - je téměř okamžitě. any skončí, jakmile najde prvek 1, který se vyhodnotí na True. all skončí hned na prvku 0, který se vyhodnocuje na False.

V následujících úlohách ti ukážeme, jak je range technicky zařízený, a jak si můžeš dělat vlastní efektivní funkce.

Autorské řešení

Řešení

def does_x_equal_y(x, y):
    return x == y

Výraz 1:

vyraz_1 = does_x_equal_y(42, 42) and does_x_equal_y(20, 50)

Výraz 2:

vyraz_2 = does_x_equal_y(46, 46) or does_x_equal_y(47, 50)

Výraz 3:

vyraz_3 = does_x_equal_y(x, 42) or does_x_equal_y(x, 50)

V každé z následujících části zaškrtni všechny funkce, které se mohou spustit.

Které z odpovědí se zavolají při vyhodnocování výrazu 1?

  • does_x_equal_y(42, 42)
  • does_x_equal_y(20, 50)

Zavolají se obě funkce, protože funkce does_x_equal_y(42, 42) se vyhodnotí na True, což zjednoduší výraz 1 na True and does_x_equal_y(20, 50), což není výraz vyhodnotitelný bez zavolání pravé části.

Které z odpovědí se zavolají při vyhodnocování výrazu 2?

  • does_x_equal_y(46, 46)
  • does_x_equal_y(47, 50)

Zavolají se pouze první funkce, protože funkce does_x_equal_y(46, 46) se vyhodnotí na True, což zjednoduší výraz 2 na True or does_x_equal_y(47, 50), což už na základě levé části lze vyhodnotit na True.

Pro které z odpovědí existuje takové x, že se při vyhodnocování výrazu 3 zavolá odpověď?

  • does_x_equal_y(x, 42)
  • does_x_equal_y(x, 50)

Pokud je x == 42, pak se vyhodnotí pouze levá část. Pokud x != 42, tak se vyhodnotí obě části.

Které z odpovědí se by se zavolali při vyhodnocování vyraz_1 and vyraz_2?

  • does_x_equal_y(42, 42) *
  • does_x_equal_y(20, 50) *
  • does_x_equal_y(46, 46)
  • does_x_equal_y(47, 50)

Z odpovědi na otázku číslo 1 víme, že se při vyhodnocování zavolají obě dvě první funkce. Výraz 1 se ale vyhodnotí na False, čímž se to zjednoduší na False and vyraz_2. Výraz 2 nemusíme již vyhodnocovat, je zjevné, že se výraz vyhodnotí na False.
