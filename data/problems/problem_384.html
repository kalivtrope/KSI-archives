Zadání

Určitě jsi si již všiml, jak moc je git super. Nicméně jak jsme si řekli v minulé úloze, git je ještě více super, když tu radost můžeš s někým sdílet. A právě k tomu slouží velké servery jako GitHub (největší server s git repozitáři, vlastněný Microsoftem) nebo GitLab (největší open-source server s git repozitáři). Po registraci si na těchto serverech můžeš založit svůj repozitář, který můžeš sdílet s celým světem, nebo jenom sám se sebou a používat jej jako zálohu svých kódů či jejich synchronizaci mezi více PC.

Také zde najdeš nepřeberné množství cizích repozitářů, ze kterých můžeš čerpat mnoho funkcionality, ať už jako knihovny pro tvoje programy, nebo přímo jako užitečné utilitky.

V této úloze po tobě budeme chtít založit si účet na GitHubu. Myslíme si, že účet na GitHubu je velice užitečná věc, kterou budeš nakonec v informatice určitě potřebovat. Nicméně, pokud bys měl něco proti zakládání účtu, můžeš tuto úlohu projít jenom teoreticky, její řešení je možné i bez registrace.

Zakládáme repozitáře na GitHubu

Aby sis mohl vytvářet vlastní repozitáře, musíš mít vytvořený vlastní účet. To uděláš jednoduše pomocí formuláře na registraci. Při registraci nezapomeň, že uživatelské jméno, které si vybereš, bude veřejně vidět a možná ho časem uvidí i tví budoucí zaměstnavatelé. Jakmile budeš zaregistrovaný, stačí kliknout na tlačítko New nebo New repository, popř. můžeš použít tento odkaz pro vytvoření tvého prvního repozitáře. Pamatuješ si náš kód na vytváření oblíbeného jídla? Bylo by jej super nasdílet s celým světem! Udělejme to tedy!

Další políčka u vytváření repozitáře

Při vytváření projektu, si můžeš vybrat, jestli chceš, aby tvůj projekt byl veřejný, nebo pouze pro tebe. Github má více různých plánů, které umí různé další věci navíc. A i GitHub premium je možné získat zdarma.

GitHub Student Pack

GitHub Student Developer Pack je obrovský balíček všemožného software, který je ti nabízen zdarma, dokud jsi student (stačí ti ISIC, email od školy nebo potvrzení o studiu). Určitě si jej prohlédni, jsou tam velice užitečné a také za normálních okolností drahé věci, od software na překládání textů až po kredit na založení vlastního VPS (virtualizované PC v cloudu), včetně právě výše zmíněného GItHub premium. No není nádherné být studentem?

V sekci Initialize this repository with ti GitHub při vytváření nabízí možnosti automatického vytvoření:

  • souboru README.md, což je soubor ve formátu MarkDown, který má obsahovat popis projektu a je zobrazen po otevření repozitáře na webu
  • vytvoření souboru .gitignore, který ti automaticky odignoruje nadbytečné soubory, pro jazyk, ve kterém chceš svůj projekt psát
  • vytvoření souboru LICENSE.txt. Ten slouží k tomu, aby ostatní vývojáři, kteří na tvůj projekt narazí, věděli, jak se k němu mohou chovat, jestli jej mohou kopírovat a používat. Pokud nevíš, jakou licenci si máš vybrat, můžeš k tomu použít tuto stránku.

Samozřejmě, tyto políčka jsou nepovinná. My je zaškrtávat nebudeme, protože my už náš projekt máme vytvořený a chceme jej pouze nahrát. Klikněme tedy na tlačítko Create repository a dokončeme tuto akci.

Nahráváme náš projekt

… pokud jej máme již vytvořený v GitHub desktop

.. tak nám stačí pouze kliknout na Publish repository a napsat název, pod kterým jej chceme na našem GitHub účtu zveřejnit. Pokud jsme se přes GitHub desktop ještě nepřihlásili do našeho účtu, bude to po nás aplikace nejdříve vyžadovat.

Další změny poté do veřejného repozitáře pošleme pomocí Repository -> Push.

… jinak to uděláme přes webové rozhraní

Když chceme nyní nahrát náš projekt na disku do tohoto nového repozitáře na GitHubu, ukazuje nám GitHub sám příkazy, které máme provést. Zajímá nás …or push an existing repository from the command line:

# nejdriv rekneme nasemu repozitari na disku, ze se ma synchronizovat s timto vzdalenm repozitarem
$ git remote add origin https://github.com/<username>/oblibene_jidlo.git
# a pote gitu rekneme, aby nas repozitar pushnul/"zatlacil" do toho vzdaleneho
# tento prikaz se nas zepta na jmeno a heslo k nasemu GitHub uctu, abychom prokazali, ze mame pravo do toho repozitare nahravat zmeny
$ git push -u origin main

Nyní, kdykoliv budeme chtít nahrát naše nové změny do vzdáleného repozitáře, stačí znovu říct gitu na našem počítači, aby změny zatlačil pomocí git push (První příkaz, ani část druhého -u origin main již nepotřebujeme, git stačilo říci pouze jednou, kam má změny posílat)

Získáváme další změny

Asi se ti zdá pouhé posílání změn z tvého počítače na servery GitHubu trochu suché. Proto zkusíme přesný opak. Stáhneme nové změny ze serverů na náš počítač. K tomu potřebujeme upravit náš projekt někde jinde, například pomocí toho, že si na GitHub webu rozklikneme soubor s kódem a budeme jej editovat přímo tam. Toho docílíme pomocí kliku na tužtičku vedle tlačítek Raw a Blame.

Na začátek souboru přidáme řádek, který vypíše, že náš kód je nyní dostupný na GitHubu, vyplníme zprávu pro commit a stiskneme tlačítko Commit to main, které uloží změny do repozitáře.

Nyní, abychom tyto změny zatáhly do repozitáře na našem počítači, spustíme git pull, který přesně toho provede. Alternativně v GitHub desktop klepneme na Fetch origin a poté na Pull origin.

Podívej se nyní do History v GitHub desktop nebo spusť git log. Vidíš tam svoji nejnovější zprávu? Stejně tak bys měl vidět změnu v souboru s kódem. Skvělé, že? Takhle můžeš spolupracovat na projektu s kamarády nebo upravovat projekt na všech svých zařízeních.

Klonujeme repozitář

Pokud si chceš poprvé stáhnout nějaký svůj repozitář nebo chceš pomoct svému kamarádovi s problémem v jeho repozitáři, budeš potřebovat si jej naklonovat na svůj počítač. To je až překvapivě snadné!

Řekněme, že by sis rád naklonoval nějakou relaxační simulaci. Co třeba Nature simulator?

K naklonování klikni na Code, což ti zobrazí pravou adresu git repozitáře, kterou si můžeš zkopírovat. Projekt poté lokálně naklonuješ pomocí příkazu git clone <adresa_repozitare>:

$ git clone https://github.com/esoadamo/NatureSimulator.git
Cloning into 'NatureSimulator'...
remote: Enumerating objects: 6, done.
remote: Counting objects: 100% (6/6), done.
remote: Compressing objects: 100% (4/4), done.
remote: Total 76 (delta 1), reused 4 (delta 1), pack-reused 70
Unpacking objects: 100% (76/76), done.

Když si nyní vypíšeš seznam všech souborů, zjistíš, že se objevila nová složka NatureSimulator, která obsahuje zdrojový kód z GitHubu.

Stejného efektu v GitHub desktop dosáhneš přes nabídku File -> Clone a repository -> URL, kam zadáš zkopírovanou adresu.

Máme hotovo!

Výborně! Nyní už umíš nejen upravovat repozitář u sebe, ale také jsi schopný si stáhnout cizí repozitář nebo nasdílet svoje úžasné nápady a projekty celému světu!
