Zadání

Víš jak matfyzák vaří vodu na čaj?

Vezme rychlovarnou konvici a podívá se, jestli je v ní voda. Když zjistí, že je prázdná, naplní ji a zapne konvici.

Pokud tam voda byla, tak ji vyleje a tím převede situaci na předchozí případ.

matfyzácké vtipy, http://www.anicka.net/mff

Syn: Kolik otoček potřebuji na zašroubování žárovky?
Otec: Pokud už je zašroubovaná, tak 0.
      Jinak ji zatoč jednou, zeptej se mě znova a k mé odpovědi přičti 1.

Rekurze je využití sebe sama. Typickým příkladem je rekurzivní definice, tedy použití X v definici X. Můžeme třeba definovat funkci faktoriál pomocí funkce faktoriál (faktoriál čísla n zapisujeme jako n!):

0! = 1
n! = n * (n - 1)! pro n > 0

Totéž slovně:

Syn: Kolik je faktoriál čísla n?
Otec: Pokud je n rovno 0, pak 1.
      Jinak se mě zeptej na faktoriál čísla (n - 1) a moji odpověď vynásob číslem n.

Nejedná se o definici v kruhu, protože v každém kroku dochází ke zjednodušování (faktoriál n je definován pomocí faktoriálu n−1) a je definován i bázový případ (n=0) bez rekurze, ke kterému vše směřuje. Jde tedy spíše o definici ve spirále. Ukážeme si to na příkladu. Kolik je třeba faktoriál čísla 4?

  • Podle druhého řádku definice je 4!=4⋅3!. Abychom tedy zjistili, kolik je 4! musíme nejprve spočítat 3!.
  • Podle druhého řádku definice je 3!=3⋅2!, takže 4!=4⋅3!=4⋅3⋅2!.
  • Podle druhého řádku definice je 2!=2⋅1!, takže 4!=4⋅3⋅2!=4⋅3⋅2⋅1!.
  • Podle druhého řádku definice je 1!=1⋅0!, takže 4!=4⋅3⋅2⋅1!=4⋅3⋅2⋅1⋅0!.
  • Podle prvního řádku definice je 0!=1, takže 4!=4⋅3⋅2⋅1⋅0!=4⋅3⋅2⋅1⋅1=24.

Faktoriál 4 je tedy 24.

V matematice můžeme využít X v definici X a podobně je to i v programování – můžeme využít funkci X uvnitř funkce X. Takovým funkcím, které volají samy sebe, říkáme rekurzivní funkce. Téměř jedna k jedné můžeme k definici matematické funkce faktoriál napsat definici výpočetní funkce factorial v Pythonu (zajímají nás jenom nezáporná celá čísla, protože pouze pro ta je faktoriál definován):

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n - 1)

Jak bude probíhat výpočet pro factorial(4)?

  • factorial(4): Podmínka n == 0 není splněná, takže by výpočet přešel na řádek return n * factorial(n - 1). Tady by došlo k rekurzivnímu volání factorial(3).
  • factorial(3): Podmínka není splněná, rekurzivní volání factorial(2).
  • factorial(2): Podmínka není splněná, rekurzivní volání factorial(1).
  • factorial(1): Podmínka není splněná, rekurzivní volání factorial(0).
  • factorial(0): Podmínka je splněná, funkce tedy vrací hodnotu 1. Teď se začneme vynořovat z rekurzivních volání.
  • factorial(1): Vracíme 1∗1, tedy 1 (ta druhá jednička je ta právě vypočtená).
  • factorial(2): Vracíme 2∗1, tedy 2.
  • factorial(3): Vracíme 3∗2, tedy 6.
  • factorial(4): Vracíme 4∗6, tedy 24.

Pokud si právě popsané chceš projít ještě jednou, interaktivně, program je pro tebe nachystaný v nástroji pythontutor.com. Na levé straně obrazovky se ukazuje zdrojový text programu, šipky ukazují na právě prováděný a na po něm následující řádek, a na pravé straně obrazovky je znázorněn zásobník volání funkcí a aktuální hodnoty proměnných. Tlačítkem Forward > posuneš výpočet o krok dále.

Jak postupovat při návrhu rekurzivních algoritmů? Nejprve určíme podproblémy, které potřebujeme k vyřešení našeho problému. (Například pro výpočet n! potřebujeme určit jediný podproblém (n−1)!. Pro vyřešení problému zašroubování žárovky n otočkami musíme vyřešit podproblém otočení žárovky (n−1) otočkami.) Tyto podproblémy vyřešíme rekurzivně a z jejich výsledků poskládáme výsledné řešení. Dále je nutné určit bázový případ (případně více bázových případů) a jejich řešení (například faktoriál 0 je 1). Bázový případ musí být takový, aby se k němu všechny větve výpočtu časem dostaly, jinak by výpočet nikdy neskončil.

Na co si při programování dávat pozor?

V ohledu k prostorové složitosti je důležité nezapomenout, že zásobník volání také zabírá paměť, a protože maximální hloubka rekurzivního zanoření bude skoro jistě záviset na velikosti vstupu, bude zásobník přispívat k prostorové složitosti. Rekurzivní funkce faktorial má lineární časovou, a také lineární prostorovou složitost.

Další důležitá informace je, že maximální hloubka zásobníku volání je omezená, tento limit je na různých počítačích v různých systémech různý, a je výrazně menší než by bylo dáno jen kapacitou operační paměti. Můj limit, v CPythonu 3.5.4 na Windows 10, je ve výchozím nastavení 1000. Limit je možné změnit pomocí funkce sys.setrecursionlimit. Kromě limitu určeného Pythonem existuje i limit daný operačním systémem, který limituje maximální velikost zásobníku volání každého procesu, v našem případě interpretru CPythonu (měřeno množstvím obsazené paměti, nikoli hloubkou zanoření). Při překročení limitu operačního systému je Python operačním systémem ukončen. Limit v Pythonu samotném je záměrně nastaven nízko, aby se minimalizovalo riziko překročení limitu operačního systému.

Jak to vypadá, když překročím kapacitu zásobníku volání? Nadefinuji funkci f() která volá sama sebe a nedefinuji žádný bázový případ, kde by se rekurze ukončila.

>>> def f():
...     f()
... 
>>> f()
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
  File "<stdin>", line 2, in f
  File "<stdin>", line 2, in f
  File "<stdin>", line 2, in f
  [Previous line repeated 996 more times]
RecursionError: maximum recursion depth exceeded

Co když si zvýším limit pomocí sys.setrecursionlimi()?

>>> import sys
>>> sys.setrecursionlimit(2**30)
>>> f()
[1]    5026 segmentation fault  python

Při dosažení limitu operačního systému je interpretr Pythonu ukončen operačním systémem. Zde 5026 je číslo procesu a segmentation fault je Linuxové jméno chyby, na Windows by se to jmenovalo “Program provedl neplatnou operaci a bude ukončen”. Segmentation fault znamená česky “chyba segmentace paměti”. Program (v tomto případě ne náš program, ale interpretr Pythonu) postupně vyčerpal celý zásobník a následně přistoupil k segmentu operační paměti, který mu nepřísluší.

Závěrem ještě něco o rekurzi pro ty z vás, kteří už znají lambda výrazy.

Kombinátor Y a dekorátory v Pythonu

Je možné definovat anonymní funkci (lambda funkci), která bude rekurzivní? Zdálo by se, že ne. Aby mohla funkce volat sama sebe, musí být schopna se na sebe odkazovat. Ale jak se na sebe může odkazovat, když nemá jméno? Řešení na způsob následujícího

f = lambda n: 1 if n == 0 else n * f(n - 1)

nelze uznat, protože ač jsem použil klíčové slovo lambda, to co jsem vytvořil je pojmenovaná funkce (jmenuje se f) a já její jméno používám, abych dosáhl rekurzivity.

Ukázalo se, že úlohu lze vyřešit, pokud si definujeme pomocnou funkci nazvanou kombinátor Y. Kombinátor je funkce jejímiž argumenty jsou jiné funkce, a která vrací funkci, která nějak kombinuje nebo vylepšuje funkce předané v argumentech. Kombinátor Y vypadá na první pohled značně neproniknutelně. Na druhý pohled stále vypadá neproniknutelně.

Autorem definice kombinátoru Y je Haskell B. Curry (1900–1982).

Y = lambda f: (lambda x: x(x))(lambda y: f(lambda n: y(y)(n)))

Funkci pro výpočet faktoriálu získám tak, že funkci Y zavolám s vhodně zkonstruovanou anonymní funkcí jako parametrem.

factorial = Y(lambda f: lambda n: 1 if n == 0 else n * f(n - 1))

Dobrý (anglický) zdroj na bližší seznámení se s kombinátorem Y je video Jim Weirich: Adventures in Functional Programming.

K čemu je to dobré v praxi? Kombinátor Y sám o sobě k ničemu, rekurzivní funkce v Pythonu se takto komplikovaným způsobem nedefinují, ale myšlenka definování funkcí tak, že se aplikuje jedna funkce na druhou, je v Pythonu hluboce zakořeněna a jazyk má dokonce speciální syntax pro zápis takových definic. Funkci, která modifikuje definice jiných funkcí, se v Pythonu říká dekorátor a Pythonová syntax pro aplikaci dekorátoru při definici funkce vypadá následovně. Všimni si znaku zavináče.

@Y
def factorial(f):
    return lambda n: 1 if n == 0 else n * f(n - 1)

Právě uvedené znamená, že dekorátor, v tomto případě funkce Y, se zavolá na námi napsanou funkci factorial, a následně se naše funkce factorial nahradí výsledkem předchozího volání.

V praxi se dekorátory například hodí, pokud chci vytvořit v Pythonu webový server pomocí knihovny Flask, jak je ukázáno v článku Webové aplikace: Flask (Nauč se Python!).

@app.route('/')
def index():
    return 'MI-PYT je nejlepší předmět na FITu!'

V prvním příkladu z článku máme funkci app.route. Zavoláme ji s argumentem '/', a návratovou hodnotu což je další funkce, aplikujeme jako dekorátor. Výsledkem je, že knihovna Flask si zaregistruje naši funkci index jako handler pro cestu '/', tedy hlavní stránku na našem webovém serveru.

Procvičení

Mějme dekorátor g, definovaný následujícím způsobem.

def g(f):
    def h(n):
        return f(n, 2) + 1
    return h

To stejné pomocí lambda výrazu.

g = lambda f: lambda y: f(y, 2) + 1

Nyní dekorátor použijeme při definici funkce m.

@g
def m(x, y):
    return x ** y

Tvým úkolem je určit, které funkce z nabízených možností odpovídají tomu, jak je definována funkce m (poté, co Python aplikoval dekorátor g).

Vyber správnou odpověď z následujících možností.

def m1(x, y):
    return x**2 + 1

def m2(x):
    return x**x

m3 = (lambda f: lambda y: f(y, 2) + 1)(lambda x, y: x ** y)
Řešení

Správně je možnost m3.

Možnost m1 ukazuje, že dekorátor může změnit počet (a jména) parametrů funkce. Dekorátor g to dělá; výsledná funkce má jen jeden parametr, ne dva. V praxi se to spíš nedělá, pro přehlednost je dobré zachovat hlavičku funkce jak ji definoval programátor.

Možnost m3 ilustruje, že velkou výhodou lambda výrazů je, že jsou to výrazy. Do výrazu g(m)lze přímo dosadit definice g a m (jak ji napsal programátor), a hned máme výsledek. Pokud znáš pravidla, jak lambda výrazy upravovat, můžeš s nimi počítat s tužkou a papírem, jako při úpravách algebraických výrazů v matematice.

Zkus vymyslet rekurzivní algoritmus, který zjistí součet přirozených čísel od 1 do n, kde n je nezáporné celé číslo. Můžeš si vymyšlený algoritmus zkusit naprogramovat v Pythonu.
